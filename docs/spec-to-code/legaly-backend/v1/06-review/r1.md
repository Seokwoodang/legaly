# 06 · 백엔드 리뷰 라운드 r1

독립 리뷰어(블라인드)가 백엔드 diff를 스펙 대비 검토. 권장대로 전부 수정.

### 🔴 R1 (블로커) — answer 함수에 외부 try/catch 없음 → 예외 시 CORS 없는 500
- 위치 `answer/index.ts`. 수정: 전체를 try/catch로 감싸 `json(500,...)`(CORS 포함). **fix ✅**

### 🟡 R2 (메이저) — legalHandler 타임아웃이 스텁 200으로 숨겨짐(504여야)
- 위치 `legalHandler.ts` catch. 수정: AbortError → 504, 그 외만 스텁 폴백. 테스트 정정(R6). **fix ✅**

### 🟡 R3 (메이저) — 로그인 실패(null)가 UI에 안 드러남
- 위치 AppContext.login + 페이지 onSubmit. 수정: AuthModal이 onSubmit 성공여부(boolean)로 닫힘/오류표시,
  실패 시 "아이디 또는 비밀번호가 올바르지 않아요" 표시. **fix ✅**

### 🟡 R4 (메이저) — getSaved가 에러를 삼켜 빈 목록으로 위장
- 위치 dataClient.getSaved. 수정: error throw → SavedPage가 catch 처리. **fix ✅**

### ⚪ R5 (마이너) — rate limit 맵 무한 증가
- 위치 answer/index.ts. 수정: 5천 초과 시 만료 버킷 정리. **fix ✅**

### ⚪ R6 (마이너) — 핸들러 테스트 라벨/커버 부정확(캐시히트를 폴백으로 오기재)
- 수정: 캐시미스+실패→스텁 200, 큐레이션 타임아웃→504, 캐시히트→fetch 미호출 테스트로 정정. **fix ✅**

### ⚪ R7 (마이너) — 비번 input에 label/aria 없음
- 위치 AuthModal. 수정: `<label for="auth-password">` 추가. **fix ✅**
