# 06 · 리뷰 라운드 r1 — 리걸리 프론트엔드

독립 리뷰어(블라인드, 스펙+그리드만 보고 코드 검토). 사용자가 각 항목을 **fix / defer(F) / reject**
로 판단. (이전 라운드 없음 → 비블라인드 closure 표 해당 없음.)

## 발견 (블라인드 디스커버리)

### 🔴 R1 (블로커) — LawsPage·CasesPage 인증 모달 제출이 login()을 호출 안 함
- 위치: `src/pages/LawsPage.tsx:106`, `src/pages/CasesPage.tsx:95`
- 문제: `onSubmit={() => setAuthOpen(false)}` — 이름만 버리고 로그인 안 됨(복붙 누락). 다른 5개
  페이지는 `(n)=>{login(n);setAuthOpen(false)}`로 정상.
- 스펙: 그리드 B1 "제출, 이름 있음 → `legaly.user={name}`".
- 수정: `useApp()`에서 `login` 가져와 `onSubmit={(n)=>{login(n);setAuthOpen(false)}}`.
- **판단: ☐ fix  ☐ defer  ☐ reject**  · *권장: fix*

### 🟡 R2 (메이저) — 상세 페이지에서 팀 전환 시 목록으로 이동 안 함
- 위치: `LawDetailPage.tsx:20–27`, `CaseDetailPage.tsx:18` (+ Header 세그먼트)
- 문제: 상세에서 팀 전환하면 같은 id로 재조회만 함. 스펙은 새 팀 목록으로 이동(stale id 회피).
- 스펙: 그리드 E2/F2 "상세에서 팀 전환 → 새 팀 `/laws`(`/cases`)로 이동".
- 수정: `prevTeamRef`로 팀 변경 감지 시 `navigate('/laws'|'/cases', {replace:true})`.
- **판단: ☐ fix  ☐ defer  ☐ reject**  · *권장: fix*

### 🟡 R3 (메이저) — 교차팀 상세 breadcrumb 클릭 시 레코드 팀을 설정 안 함
- 위치: `LawDetailPage.tsx:39`, `CaseDetailPage.tsx:34`
- 문제: 교차팀 레코드(예: 법무 컨텍스트에서 소득세법)에서 breadcrumb "세무팀 자료실" 클릭 시
  `legaly.team`을 세무로 안 바꾸고 `/laws`로 가 → 법무 목록이 뜸.
- 스펙: 확정 스펙 §I "교차 팀 상세 breadcrumb 클릭 → `legaly.team = 레코드 팀` 설정 후 이동".
- 수정: breadcrumb `onClick`에서 `if (record.team !== team) setTeam(record.team)`.
- **판단: ☐ fix  ☐ defer  ☐ reject**  · *권장: fix*

### 🟡 R4 (메이저) — getSaved가 team 미상 항목을 버림(법무 팔레트로 보정해야 함)
- 위치: `src/lib/storage.ts:35–45`
- 문제: `isValidSaved`가 team이 법무/세무 아니면 항목 전체를 드롭. 스펙은 필수필드 누락만 제거하고
  team 미상은 법무 팔레트로 보정·유지. 구 스키마 저장분이 통째로 사라질 수 있음.
- 스펙: 확정 스펙 §아키텍처/그리드 A1 "미상 team → 법무 팔레트로 렌더".
- 수정: 필수필드 검증과 team 보정을 분리(필드 있으면 유지, team 비정상 시 '법무'로 map).
- **판단: ☐ fix  ☐ defer  ☐ reject**  · *권장: fix*

### 🟡 R5 (마이너) — AuthModal 닫을 때 트리거로 포커스 복귀 안 함
- 위치: `src/components/AuthModal.tsx:22–24`
- 문제: 닫힘 시 포커스가 body로 — WCAG 및 스펙 위반.
- 스펙: 그리드 B1 "닫을 때 트리거로 포커스 복귀".
- 수정: open 시 `document.activeElement` 저장, close 시 복귀.
- **판단: ☐ fix  ☐ defer  ☐ reject**  · *권장: fix*

### ⚪ R6 (닛) — 팀 전환 리셋 effect에서 `sending`을 안 지움
- 위치: `src/pages/ConsultPage.tsx:42–46`
- 문제: 인플라이트 중 팀 전환 시 `sending=true` 잔류 → 실제 비동기 백엔드에서 입력/전송 비활성
  고착 가능(현재 스텁은 즉시 resolve라 비가시).
- 스펙: 그리드 §I "인플라이트 중 팀 전환 → 깨끗한 인사말 표시".
- 수정: 리셋 effect에 `setSending(false)` 추가.
- **판단: ☐ fix  ☐ defer  ☐ reject**  · *권장: fix*
