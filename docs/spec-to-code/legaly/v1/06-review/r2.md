## (a) 블라인드 신규 지적
> 리뷰어가 r1에서 false-candidate로 올렸던 다수 항목은 자체 철회(clean). 아래만 실제 이슈.

### 🔴 N1 (블로커) — 로비 보관함 미리보기가 첫 마운트 후 갱신 안 됨
- 위치: `src/pages/LobbyPage.tsx:21` — `useEffect(()=>{getSaved...},[])`
- 문제: 스펙 §I "로비 방문 시마다 saved 재읽기". dep `[]`라 상담 저장/삭제 후 로비 복귀 시 stale.
- 수정: `useLocation()` + dep `[location.key]`.
- **판단: ☐ fix  ☐ defer  ☐ reject** · *권장: fix*

### 🔴 N2 (블로커) — 보관함 필터가 방문마다 '전체'로 리셋 안 됨
- 위치: `src/pages/SavedPage.tsx:20,23` — filter는 첫 마운트 초기값만, location.key 재방문 시 유지.
- 문제: 스펙 §I "필터는 방문마다 전체로 리셋".
- 수정: location.key effect에서 `setFilter('전체')` 함께.
- **판단: ☐ fix  ☐ defer  ☐ reject** · *권장: fix*

### 🟡 N3 (메이저) — "이어서 상담하기" 2회차부터 자동전송 누락 (실제 기능 버그)
- 위치: `src/pages/ConsultPage.tsx:50–62` — `sentPendingRef`가 한 번 true가 되면 영구 true.
- 문제: pendingQ로 상담 진입(자동전송) → 보관함 "이어서 상담하기"로 재진입(새 location.key + 새
  pendingQ)해도 `sentPendingRef.current===true`라 즉시 return → **두 번째 질문이 조용히 드롭**.
- 수정: location.key 단위로 가드를 effect-로컬(`let fired`)로 바꿔 매 네비게이션마다 재활성
  (StrictMode 안전은 cleanup의 clearTimeout + 로컬 가드로 유지). `sentPendingRef` 제거.
- **판단: ☐ fix  ☐ defer  ☐ reject** · *권장: fix*

### 🟡 N4 (메이저, 현재는 스텁이라 잠복) — 상세 페이지 비동기 fetch 언마운트 가드 없음
- 위치: `LawDetailPage.tsx:29–33`, `CaseDetailPage.tsx:27` — getLaw/getCase `.then(set...)`에 alive 가드 없음.
- 문제: fetch 중 이탈 시 언마운트 후 setState(현재 즉시 resolve 스텁이라 비가시, 실제 API 연동 시 표면화).
  스펙 §I "언마운트 후 setState 없음"은 상담만 적용돼 있고 상세엔 누락.
- 수정: effect에 `let alive=true; return ()=>{alive=false}`; 각 set 전 `if(!alive)return`.
- **판단: ☐ fix  ☐ defer  ☐ reject** · *권장: fix (백엔드 연동 전 필수)*
