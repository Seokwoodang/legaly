### A1. localStorage / 영속화(결정됨)
| 키 | 읽기 | 쓰기 | 비정상/손상 → |
|---|---|---|---|
| `legaly.team` | 모든 팀 페이지 마운트 | 팀 세그먼트 / 로비 카드 클릭 | 법무/세무 아님 → `법무` ✅ |
| `legaly.user` | 모든 페이지 마운트 | 인증 제출; 로그아웃 시 제거 | JSON 깨짐 → `null` ✅ |
| `legaly.saved` | `/saved`, 로비 미리보기 | `/consult` 저장, `/saved` 삭제 | JSON 깨짐 → `[]`; 항목 team 비정상 → `법무` 팔레트로 렌더 ✅ |
| `legaly.pendingQ` | `/consult` 마운트 | 로비 칩, `/saved` 이어서 | 없음 → 자동전송 안 함 ✅ |

- **접근 경계** 🆕: 4개 키 모두 타입 지정 `storage` 모듈(키 상수 + 타입 get/set) 경유; saved CRUD는
  `dataClient`가 `storage` 사용. 컴포넌트는 `localStorage` 직접 접근 금지.
- **dataClient 동기/비동기** 🆕: **전부 비동기**(`Promise<T>`); localStorage 스텁은 즉시 resolve.
  백엔드 교체 시 컴포넌트 무수정 위해.

### D2. /consult 팀 전환
| 상태 | 전환 후 |

- `getAnswer(team,q)`는 이번 작업에서 **q 무관하게 팀 고정 샘플 반환** ✅(스텁은 q 로깅; 백엔드가

  `/laws/:id` & `/cases/:id` 링크용 `id` 포함 🆕.

- 출처 카드 링크: 법령 전문 보기 → `/laws/{citation.id}`; 판례 상세 보기 → `/cases/{citation.id}` 🆕.

## I. 비평가 보완 추가분(Phase 2 완전성 패스)

적대적 완전성 비평가가 찾은 빈틈을 각각 메움(3개 ⚠️확인 항목은 영향 없음).

**콜드 스타트 / 영속화**
- pendingQ 존재하나 공백(`""`/공백) → 없음으로 처리, 자동전송 안 함. ✅
- `legaly.team` **없음**(첫 방문)과 **비정상** 모두 → `법무`. ✅
- 소스 번들 = **페이지 `.dc.html` 7개 + `support.js`**(런타임, 이식 대상 아님). (서두 개수 정정)

**헤더 / 셸**
- `메뉴 열림` 상태는 로그인 시에만 도달 가능; `(메뉴 열림, 비로그인)`은 불가능.
- 인증 모달은 비로그인 시에만 오픈; 컴포넌트가 `openAuth` 가드(로그인 상태면 no-op).
- 인증 모달 열림 중 팀 전환 → 아래에서 팀 전환, **모달은 유지**.
- `/saved` 서브내비 활성 탭 색 = **현재 팀 accent**(팀 상태 따라감).
- 로비 카드 링크는 `legaly.team`을 **클릭 핸들러에서 동기적으로(이동 전)** 기록(storage.set 동기);
  목적지가 올바른 팀으로 마운트.

**로비**
- 제목 "내 보관함" + "전체 보기 →"는 **항상 렌더**; 아래 내용(카드 vs 점선 빈)만 조건부.
- 미리보기 = `saved.slice(0,3)` 원본 배열 순서(최신순), 재정렬 없음.

**상담 — 인플라이트 & 비동기 생애주기**
- 인플라이트 인디케이터 메시지: `savable:false`, 저장 버튼 없음; 인플라이트 중 추천칩 & 전송 비활성.
- **인플라이트 중 팀 전환** → 보류 결과 폐기(resolve 시 무시); 깨끗한 인사말 표시.
- **인플라이트 중 이탈** → 언마운트 시 결과 무시(isMounted/AbortController ref); 언마운트 후 setState 없음.
- **보류 저장 + 팀 전환 경합** → `pendingSave`가 저장 클릭 시점 **팀 스냅샷** 보관; doSave가 그걸 사용.
- 인사말 메시지: `savable:false`(명시).
- `SavedItem.id = crypto.randomUUID()`(단일 권위 체계; 스펙 §2 갱신).

**법령/판례 상세 — 진입점 & 교차 팀**
- **콜드 스타트 딥링크** `/laws/:id`·`/cases/:id`(팀 미설정): `legaly.team` 기본 `법무`; id 우선 교차
  팀 레코드 표시돼도 헤더 세그먼트는 그 기본값 표시(accent는 레코드 따라감).
- 교차 팀 상세에서 breadcrumb 클릭 → `legaly.team = 레코드 팀` 설정 후 →`/laws`(또는 `/cases`).
- 참조조문 칩 & 관련 판례 링크 → `/laws/:id`÷`/cases/:id` 단순 이동(id 우선), **팀 쓰기 없음**.
- 상세에서 `switchTeam`은 동기 이동(이전 레코드 인플레이스 재렌더 없음).
- **뒤로/앞으로 이동**: 목록 페이지는 재마운트하며 **필터 상태 리셋**(field/query/sort/court 기본값;

  트리거 안 함; 특정 조문 직접 딥링크 없음.

**판례 스텁 데이터 형태**
