# 04 · 테스트 문서 — 리걸리 프론트엔드 (RED)

테스트 우선(RED) 명세. QA 엔지니어가 **테스트 코드를 열지 않고도** 읽고 손으로 재검증하고, 빠진
케이스를 그리드에 역제안할 수 있도록 작성. 모든 항목은 `00-behavior-grid.md` 셀에 매핑됨.

> **RED 실행 시점(중요):** spec-to-code 게이트 가드는 `testsApproved=true` 전까지 **구현/설정 파일
> 쓰기를 차단**하므로(테스트 파일만 허용), 테스트 러너 스캐폴딩과 RED 실행은 **테스트 승인 직후

## 🔍 수동 QA 절차 (앱 실행 후 손으로 재검증)
1. `npm run dev` → 로비. 로그인 안 한 상태에서 "전세금을 못 받고 있어요" 칩 클릭 → 상담 페이지에서
   질문이 자동 전송되고 법무팀 답변 + "근거 3건 접기"가 보이는지.
2. 답변의 "이 상담 저장하기" → 인증 모달(저장 변형 문구) → 이름 입력 → 토스트 "보관함에 저장했어요" →
   보관함 탭에서 항목 확인.
3. 헤더에서 세무팀으로 전환 → 대화가 세무팀 인사말로 리셋되는지, 색이 초록으로 바뀌는지.
4. 자료실 → "민법"(전문 없음) 클릭 → "전문은 준비 중입니다" 스텁. "주택임대차보호법"(전문) 클릭 →
   조문 목차 클릭 시 해당 조문으로 스크롤.
5. 판례 → 법원 드롭다운 "대법원 전원합의체" → 전합 사건만. 사건번호로 검색.
6. 보관함 → 법무팀/세무팀 필터, 삭제, "이어서 상담하기" → 상담에서 그 질문 자동 전송.
7. 로그아웃 → 보관함 항목이 유지되는지(전역 공유 정책).
8. 모바일 폭(375px)에서 가로 스크롤 없이 1열로 접히는지, 상세 사이드바가 본문 위로 가는지.

## QA가 더 의심해볼 변형 (그리드 역제안 후보)
- 같은 답변을 새로고침 후 다시 저장 시 보관함 중복(현재 허용 — 정책 확인).
- 상담 인플라이트 중 팀 전환/페이지 이탈 시 유령 응답이 안 붙는지(비동기 가드).
- pendingQ가 남은 채 상담 외 페이지 새로고침 → 다음 상담 진입 시 1회만 전송되는지.
- 교차팀 딥링크(`/laws/income-tax`를 법무 컨텍스트에서) accent/breadcrumb가 레코드 팀을 따르는지.
- localStorage 꽉 찼을 때 저장/삭제 실패 토스트 및 비관적 복구.
- 매우 긴 사용자 이름/이모지 이름의 아바타 이니셜.
