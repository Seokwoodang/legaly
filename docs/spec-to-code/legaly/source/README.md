# Handoff: 리걸리 — "나만의 법무팀 & 세무팀" (Legaly)

## Overview
리걸리(Legaly)는 일반 사용자가 **자기 전용 법무팀·세무팀**을 둔 것처럼 느끼게 하는 한국 법률·세무 종합 서비스입니다. 한 서비스 안에 **법무팀(법률)**과 **세무팀(세무)** 두 축이 있고, 각 팀에 들어가면 곧바로 **AI 상담**이 시작됩니다. 팀이 사용자를 이름으로 맞이하고, 질문하면 관련 **법령 조문**과 **판례**를 근거로 답합니다. 자료(법령) 검색과 판례 검색은 팀 안의 보조 탭입니다. 상담 결과는 로그인한 사용자의 **보관함**에 저장해 이어볼 수 있습니다.

핵심 컨셉 3가지:
1. **팀 = 상담 우선** — 팀에 들어가면 검색이 아니라 대화가 먼저. 자료실/판례는 팀 내부 보조 탭.
2. **개인화 + 로그인** — 이름으로 로그인하면 전담 팀이 이름으로 맞이하고 상담을 저장.
3. **친근하지만 신뢰감 있는 톤** — 따뜻한 뉴트럴 배경 + 대화체 카피, 신뢰감(네이비)은 절제해서 강조에만 사용.

## About the Design Files
이 번들의 `design_files/*.dc.html` 파일들은 **HTML로 만든 디자인 레퍼런스(프로토타입)**입니다. 의도한 모양·동작·흐름을 보여주는 것으로, **그대로 복사해 배포하는 프로덕션 코드가 아닙니다.** 작업 목표는 이 디자인을 **대상 코드베이스의 기존 환경(React/Next, Vue, SwiftUI, native 등)과 패턴·라이브러리에 맞춰 다시 구현**하는 것입니다. 아직 환경이 없다면 프로젝트에 가장 적합한 프레임워크를 선택해 구현하세요(권장: React + TypeScript + Tailwind 또는 기존 디자인 시스템).

> `.dc.html` 파일은 자체 미니 런타임("Design Component")으로 동작합니다. 각 파일은 인라인 스타일 템플릿 + `class Component extends DCLogic { renderVals() {...} }` 로직으로 구성됩니다. `support.js`는 그 런타임이며 **구현 시 이식 대상이 아닙니다** — 참고용으로만 동봉했습니다. 로직 클래스의 상태/핸들러/데이터는 그대로 읽어 의도를 파악하세요.

## Fidelity
**High-fidelity (hifi)** — 최종 색상·타이포그래피·간격·인터랙션·카피가 모두 확정된 픽셀 정밀 목업입니다. UI는 코드베이스의 기존 라이브러리/패턴으로 **픽셀에 가깝게** 재현하세요. 한국어 카피는 그대로 사용하시면 됩니다.

## 정보 구조 (Information Architecture)

두 단계 내비게이션입니다.

**① 로비 (홈, 비‑팀 컨텍스트)** — `index`
- 헤더: 로고 + 계정(로그인/사용자). 팀 탭/서브내비 없음.
- 본문: 인사 + "내 법무팀 / 내 세무팀" 두 카드(각 팀으로 진입) + 보관함 미리보기.

**② 팀 내부 (팀 컨텍스트)** — `ai`, `laws`, `law-detail`, `cases`, `case-detail`, `saved`
- 헤더 1행: 로고 + **팀 전환 세그먼트(법무팀/세무팀)** + 계정.
- 헤더 2행: **서브 내비 탭 = 상담 · 자료실 · 판례 · 보관함** (현재 팀 색으로 활성 표시).
- 현재 팀은 모든 팀 페이지에서 공유되는 전역 상태(아래 "State / 영속화" 참조).

페이지 ↔ 라우트 매핑(권장):
| 디자인 파일 | 의미 | 권장 라우트 |
|---|---|---|
| `index.dc.html` | 로비/대시보드 | `/` |
| `ai.dc.html` | 팀 상담(채팅) | `/consult` |
| `laws.dc.html` | 팀 자료실(법령 목록) | `/laws` |
| `law-detail.dc.html` | 법령 상세 | `/laws/:id` |
| `cases.dc.html` | 팀 판례 목록 | `/cases` |
| `case-detail.dc.html` | 판례 상세 | `/cases/:id` |
| `saved.dc.html` | 보관함(저장된 상담) | `/saved` |

> 디자인에서는 팀이 전역 상태라 라우트에 팀이 안 들어가 있지만, 실제 구현 시 `/law/consult`, `/tax/consult` 처럼 팀을 라우트에 넣어도 좋습니다.

## State Management & 영속화 (중요)

디자인 프로토타입은 모든 영속 상태를 **localStorage**에 저장합니다. 실제 앱에서는 인증/DB/상태관리로 대체하되, 키 의미는 동일하게 유지하세요.

| localStorage 키 | 값 | 의미 / 실제 구현 대응 |
|---|---|---|
| `legaly.team` | `"법무"` \| `"세무"` | 현재 선택된 팀(전역). 모든 팀 페이지가 마운트 시 읽음. → 전역 상태(Context/store) 또는 라우트 파라미터 |
| `legaly.user` | `{"name": string}` JSON | 로그인 사용자(데모). → 실제 인증 세션/유저 객체 |
| `legaly.saved` | `Array<SavedItem>` JSON | 저장된 상담 목록. → DB 테이블 `saved_consultations` (user별) |
| `legaly.pendingQ` | `string` | 다른 페이지에서 넘어올 때 자동 전송할 질문(딥링크). → 라우트 state/쿼리 파라미터로 전달 |

`SavedItem` 형태:
```ts
type SavedItem = {
  id: string;        // 고유 id (디자인: messageId + '_' + Date.now())
  team: "법무" | "세무";
  title: string;     // 저장 대상 답변을 유발한 사용자 질문
  summary: string;   // AI 답변 첫 부분(개행 제거, 120자 + '…')
  date: string;      // "YYYY. M. D."
};
```

**페이지별 핵심 state**
- 공통(모든 팀 페이지): `team`, `user`, `authOpen`(로그인 모달), `menuOpen`(계정 드롭다운), `authName`(모달 입력값).
- `ai`(상담): `messages[]`, `draft`(입력중), `expanded{messageId:bool}`(근거 펼침), `savedIds{messageId:bool}`(저장됨 표시), `toast`(저장 알림 문구).
- `laws`/`cases`: `field`(분야 칩), `query`(검색), `sort`('name'|'date' / 'new'|'old'), (cases) `court`.
- `law-detail`: `active`(현재 조문 id, 목차 하이라이트).
- `saved`: `filter`('전체'|'법무'|'세무'), `saved[]`.

**상태 전이 트리거**
- 팀 전환(세그먼트 클릭): `legaly.team` 갱신 + 페이지 콘텐츠 재렌더. 상담 페이지에서는 **대화를 그 팀의 인사말로 리셋**. 자료실/판례는 분야 칩을 '전체'로 리셋.
- 로그인 제출: `legaly.user` 저장 → 헤더·인사·게이팅 갱신. 상담 페이지에서 "저장하려다 로그인" 상황이면 로그인 직후 **보류된 저장을 실행**.
- 상담 저장: 로그인 상태면 `legaly.saved`에 unshift + "저장됨"으로 버튼 전환 + 토스트(2.6초). 비로그인 상태면 로그인 모달 오픈(`pendingSave` 보관 후 로그인 성공 시 저장).
- 딥링크 질문: 홈 카드의 예시 칩/보관함 "이어서 상담하기" 클릭 → `legaly.team`+`legaly.pendingQ` 저장 후 `ai`로 이동. `ai` 마운트 시 `pendingQ`가 있으면 제거 후 자동 전송.

### 2. 팀 상담 — `ai.dc.html`  (전체 높이 100dvh, 본문 max-width 760, bg #f1efea)

- 새 메시지 추가 시 채팅 영역 맨 아래로 스크롤(`scrollTop=scrollHeight`, **scrollIntoView 사용 금지**).

## Interactions & Behavior 요약
- **네비게이션:** 모든 페이지 이동은 일반 링크. 팀 컨텍스트는 `legaly.team`으로 유지(이동 전 클릭 핸들러에서 설정).
- **팀 전환:** 같은 페이지 내 즉시 재렌더(리로드 없음). 색/데이터/카피 전부 팀에 맞게 전환.
- **상담 전송:** Enter 전송, Shift+Enter 줄바꿈, 빈 입력 무시. 전송 시 사용자 메시지 + AI 답변(+출처) 추가, 자동 스크롤.
- **근거 펼침:** AI 답변별 토글. 새 답변은 기본 펼침 상태로 추가.
- **저장:** 로그인 게이팅 + 토스트 + 버튼 상태 전환. 저장 항목은 `saved`/대시보드에 반영.
- **검색/필터/정렬:** 클라이언트 측 즉시 필터(디자인은 정적 배열). 실제는 API 쿼리로 대체.
- **반응형:** 데스크톱 다열 → 모바일 1열, 상세 사이드바는 모바일에서 본문 위로. 모든 인터랙티브 타깃 ≥ 44px 권장.
- **hover:** 카드 보더 진해짐 + 살짝 상승/그림자, 버튼/칩 미세 변화(각 파일 `style-hover` 참고).

## 구현 권장 스택 / 메모
- 권장: React + TypeScript, 라우팅(React Router/Next), 상태(전역 team/user는 Context 또는 Zustand), 스타일(Tailwind 또는 기존 디자인 시스템 — 위 토큰을 테마로 매핑).
